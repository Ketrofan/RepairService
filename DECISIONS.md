# DECISIONS.md

Ключевые решения (5–7 пунктов) по проекту “Заявки в ремонтную службу”.

1. **Docker Compose + MySQL 8.0**  
   Выбран предпочтительный вариант запуска по ТЗ: `docker compose up -d`, приложение доступно локально без VPS.

2. **Простая авторизация через стандартный Laravel session guard**  
   Логин по email/password, пользователи создаются сидером. Это достаточно для проверки ролей.

3. **Роли через `UserRole` enum + `users.role`**  
   Минимальная RBAC: dispatcher/master. Ограничение доступа сделано middleware `role:*`.

4. **Статусы заявки через `RequestStatus` enum**  
   Разрешённые статусы строго ограничены (`new/assigned/in_progress/done/canceled`). Это уменьшает риск “невалидных” переходов.

5. **Защита от гонки “Взять в работу” атомарным UPDATE**  
   Логика take-in-work реализована как `UPDATE ... WHERE status='assigned' AND assigned_master_id=...`.  
   Это гарантирует: один запрос обновит строку, второй получит 0 affected rows → возвращаем 409.

6. **UI на Blade (без фронтенд-фреймворков)**  
   Цель — минимальная сложность, быстрый ручной QA и простота запуска/проверки.

7. **Валидация через FormRequest + понятные ошибки в UI**  
   Все поля заявки обязательны.  
   ФИО ограничено буквами (Unicode, пробел, дефис).  
   Телефон строго `+7` + 10 цифр, автоподстановка `+7` на форме.

8. **Feature-тесты как основной минимум регрессии**  
   Тесты проверяют ключевые сценарии на уровне HTTP: назначение мастера и 409 на повторный take.  
   Учитывали, что тестовая БД по умолчанию SQLite `:memory:`, поэтому избегаем MySQL-only миграций.
